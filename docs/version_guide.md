# バージョン管理ガイドライン

形式：`MAJOR.MINOR.PATCH`

## 更新基準

### MAJOR (メジャー)
サイトの根本的な仕組みやデザインが大きく変わる場合に更新する。
- 全体的なデザインの大幅なリニューアル
- 使用フレームワークのメジャーアップデート（例：Next.js 14 -> 15）
- ディレクトリ構造やルーティングの全面的な再設計

### MINOR (マイナー)
新しい機能やページが追加された場合に更新する。
- 新しいページの追加（例：ブログ機能、ポートフォリオ詳細ページ）
- 新しい機能の実装（例：ダークモード、検索機能、お問い合わせフォーム）
- 既存機能の目に見える改善

### PATCH (パッチ)
既存のものの修正や、小さな調整の場合に更新する。
- 誤字脱字、コンテンツの文章修正
- 軽微なCSSの調整（マージンや色の微調整）
- 依存パッケージの更新（セキュリティパッチなど）
- バグ修正

---
**メモ:**
迷った場合は「MINOR（マイナー）」を上げる。
